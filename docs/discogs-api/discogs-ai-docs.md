# Discogs AI Documentation Map

This document serves as a comprehensive map for AI agents to navigate and understand the Discogs documentation structure. It provides a brief description of each file's content to facilitate efficient information retrieval.

## 1. Core Documentation

*   **`discogs/docs/authentication.md`**: **Authentication**. This documentation explains how to authenticate with the Discogs API, detailing the registration process for obtaining consumer keys/secrets or user tokens and outlining the Discogs Auth and OAuth flows for making authenticated requests. It clarifies the different authentication methods and their implications for rate limiting, image access, and user-specific data.
*   **`discogs/docs/database.md`**: **Database**. This documentation describes how to retrieve "Release" data, which represents physical or digital objects by artists, from a database. It details the GET request for a specific release using its ID and optional currency, and provides an example response containing comprehensive release information.
*   **`discogs/docs/discogs-api-documentation.md`**: **Discogs Api Documentation**. This documentation provides information and resources for developers to interact with the Discogs database programmatically using its API. It outlines how to access and utilize the Discogs API for various data retrieval and submission tasks.
*   **`discogs/docs/home.md`**: **Home**. This documentation introduces the Discogs API v2.0, a RESTful interface for accessing and managing Discogs music data. It explains how developers can use the API to build applications, outlining data access, usage terms, quickstart examples, and essential guidelines like providing a unique User-Agent string and understanding rate limits.
*   **`discogs/docs/images.md`**: **Images**. This documentation explains the "Image" resource, which represents user-contributed images for database objects like Artists or Releases. It details how to access these images via fully-qualified URLs provided in other resource responses, requiring authentication and being subject to rate limiting.
*   **`discogs/docs/index.md`**: **Index**. This documentation provides a guide to the Discogs API v2.0, a RESTful interface for developers to build applications utilizing Discogs music data. It covers how to access information on artists, releases, and labels, manage user collections, and create marketplace listings, while also outlining usage policies, quickstart instructions, and rate limiting details.
*   **`discogs/docs/inventory-export.md`**: **Inventory Export**. This documentation details the API endpoints for managing inventory exports, including requesting a new CSV export, retrieving a list of recent exports, checking the status of a specific export, and downloading the exported file. It outlines the necessary HTTP methods, request parameters, and expected responses for each operation.
*   **`discogs/docs/inventory-upload.md`**: **Inventory Upload**. This documentation explains how to add or change inventory listings by uploading a CSV file. It details the required file structure, field names (including required and optional fields), and valid condition values for both media and sleeve.
*   **`discogs/docs/marketplace.md`**: **Marketplace**. This documentation describes the API endpoint for retrieving a user's marketplace inventory, detailing how to fetch listings with various filtering and sorting options. It explains the differences in returned information based on user authentication and provides an example API call and response structure.
*   **`discogs/docs/user-collection.md`**: **User Collection**. This documentation details the Discogs API for managing a user's collection, which is organized into folders. It explains how to retrieve, create, and get metadata for these folders, including special "All" and "Uncategorized" folders, and notes that each release instance within a folder has a unique ID.
*   **`discogs/docs/user-identity.md`**: **User Identity**. This documentation describes two API endpoints for retrieving user information: `/oauth/identity` for basic authenticated user details, and `/users/{username}` for a more comprehensive user profile including collection and wantlist data. It explains how to use these GET requests, their parameters, and provides example responses for both successful and unauthorized access.
*   **`discogs/docs/user-lists.md`**: **User Lists**. This documentation describes how to retrieve user lists and the items within a specific list using the Discogs API. It details the GET endpoints, required parameters (username or list ID), and example JSON responses for both operations, noting that private lists are only visible to the authenticated owner.
*   **`discogs/docs/user-wantlist.md`**: **User Wantlist**. This documentation describes the "Wantlist" API resource, which allows users to retrieve a list of releases a specific user wants. It details how to use the GET request with a username parameter to fetch basic information about wanted releases, noting privacy considerations and the visibility of "notes" for the wantlist owner.
